# draw.io Practical Guide

## 1. Pre-Diagram Preparation

- 先に対象読者を決める。
- 先に図の目的を 1 文で決める。
- ノード一覧と接続一覧を先に作る。
- 1 枚に収まらない場合は図を分割する。

## 2. Layout Selection

- `Horizontal flow`: 左から右へ時系列で読む図。
- `Vertical flow`: 上から下へ段階的に読む図。
- `Tree`: 親子構造や責務分解。
- `Organic`: 密なネットワーク関係の可視化。
- `Circle / Radial`: 中心要素からの展開。

レイアウト適用後は線交差と視線移動を確認し、必要なら手動で整える。

## 3. draw.io Editor Tips

- `Page View` を有効化し、印刷範囲や余白を意識する。
- 接続点やグリッドを表示し、整列コストを下げる。
- よく使うスタイルはテンプレート化して再利用する。
- 余白と文字サイズを先に決めてから要素を増やす。

## 4. XML / LLM-Specific Notes

- `mxGraphModel` に `dx` / `dy` を明示し、座標のぶれを減らす。
- テキスト要素ごとに `fontFamily` を明示する。
- 矢印セルを背面にし、ノードやラベルの可読性を守る。
- 生成結果は PNG でも確認し、重なりや欠けを検出する。

## 5. Review Checklist

- 読み順が一方向で追えるか。
- 主要コンポーネントにラベルがあるか。
- 交差線が最小化されているか。
- 色、線種、記号の意味が一貫しているか。
- 図の目的に対して不要要素が混入していないか。
